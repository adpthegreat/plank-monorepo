# Contribution Guidelines

Thank you for your interest in contributing. Some key guidelines:

## Issue Assignment & Opening PRS
Comment on the issue and wait to be assigned before opening a PR.

You may run ahead and open a draft PR but you risk having your issue closed if
someone was already assigned first or a core contributor happened to be working
on it already.

Before requesting a review make sure to run the tests, linter and formatter
locally. In plankc you can do this by running `just ci-all`.

## LLMs & AI

You must disclose any use of LLMs / AI. Even when using LLMs you are fully
responsible for its output. AI code must be scrutinized more heavily due to its
tendency to write things that look right but aren't.

Slop PRs will have your issue instantly closed and have you potentially banned.
